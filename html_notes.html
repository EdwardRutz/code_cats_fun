HTML NOTES

Sources: TeamTreehouse Front-End Development Track, Mozilla Developer Network

  • Displaying < HTML > Tags In-line - Use < and & gt to replace the < and > so a tag shows up in text (< img >). The & lt means "less than" and & gt means "greater than." They're used to explicitly define less than and greater than symbols. If one wanted to type out < html > and not have it be a tag in the HTML, one would use them. An alternate way is to wrap the < code > element around code to not run into that. They can also be used to present mathematical operators.
  • Open a Twitter Window - This code will open up a Twitter window for the user to send a tweet, replace "edwardr" with your Twitter username (e.g. < a href="http://twitter.com/intent/tweet?screen_name=edwardr" > Twitter: @edwardr < /a >).
  • HTML Character Entities - Entities replace all the characters that are used by HTML such as <; or >;. Also included are various symbols not on the keyboard like © or €. And entity names are case sensitive. For a list of entities, see W3 Schools: HTML Character Entities .

HTML ELEMENTS

  • For a list of HTML Elements, see W3.org: List of HTML Elements
  • A quick cheatsheet for HTML, Javascript, CSS, etc. See W3.org Cheat Sheet
  • <br> In HTML there is no end tag. XHTML must be closed (e.g. <br />). The tag inserts a single line break. Use to enter line breaks and not to seperate paragraphs. It is useful for writing poems or addresses (Source: W3 Schools).
  • <code> - The code tag is a phrase tag used to define computer code. It will format the text in the monospace font.
  • <img> - The image element is a self closing element with two required attributes. The source (src) attribute points to the image file and the alternate (alt) attribute describes the image using text. You don't have to close the <img> tag in HTML, because it doesn't have any content (the image is only referenced in the tag's attributes). They are called empty elements, just like the <br> tag.
  • <link> - The link element connects other files to an HTML document, such as CSS or JavaScript files.
  • <span> is a generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang. It should be used only when no other semantic element is appropriate. <span> is very much like a <div> element, but <div> is a block-level element whereas a <span> is an inline element.Source: MDN <span>
  • <style> - The style element can contain CSS code that will be applied to the HTML. Typically it's better to keep CSS in an external stylesheet.
