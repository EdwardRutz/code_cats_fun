CSS NOTES

Source: TeamTreehouse, "Front-End Development Track"

  • CLASS vs ID: The difference between a CSS class and ID? Classes are reused on the site, IDs are unique and are used only once. On the CSS page style Classes starting with a period (.className) and IDs start with a hashmark (#idName). "Classes are used when you want to manipulate multiple elements at a time and id's are used when you want to manipulate a single element. You can have multiple classes but you can only have 1 id (Isaiah McGee)
    • "Much like student ID cards are distinct. No two students on campus will have the same student ID card. However, many students can and will share at least one Class with each other." For example:
    • < student id="jonathanSmith" class="biology" >

      < student id="maryJones" class="calculus" >

    • Differences Between Class and IDs
    • IDs carry more weigth than classes because they refer to something very specific within the element.
    • Classes are used for global attributes, it is recommended to avoid using ID's, unless it is absolutely necessary.
    • It is good to keep functionality and presentation of code separate, so use IDs for fragment identifiers and JavaScript hooks and use classes for your over-all styling. The benefits of this approach are that classes are more versatile because they can be reused, this makes your code more modular.
    • Using IDs and classes for separate purposes avoids any potential specificity issues and makes your code easier to understand by another developer, which is important if you're working as part of a team.
  • Using IDs in Selectors: - Should we use IDs in Selectors? There are several opinions...

    • Why Shouldn't I Use ID Seletors in CSS?
    • Don't Use ID Selectors in CSS
    • Don't Use IDs in CSS Selectors
    • Disallow IDs in Selectors

  • DESCENDENT SELECTORS: CSS lets us target elements based on their relationship in the HTML document. We can combine selectors to create descendant selectors that target elements that are descendants of an element. This makes our selectors more specific.
    • Descendant Selectors, or Combinators, describe the descendant of an element and can be used for styling elements that exist inside of other elements.
  • Google Fonts - A 3rd party font service from Google that provides a large library of fonts that are legitmately licensed. All of the fonts are available for free.
  • Inheritance - When several values are given, the most specific value will be applied. The child style overights the parent style.
  • Remembering Top, Right, Bottom, Left Values - The values are ordered clockwise starting at the Top, then Right, then Bottom and finally Left (e.g. padding: 5px 0 0 0;) Remember to put a space between the values.
  • Moving Up Directories/Folders - using ".." will move up a directory. For example, in the css dir move to img dir by typing "../img/phone.png". So we go up one to the root directory then follow the path to the img directory and phone.png.
  • USE PERCENTAGES as values to define element size instead of using pixels. It makes the website more fluid and responsive to different sizes of displays. Percentages are relative units that will change based on the relative width of the browser. Fixed units like pixels stay the same size regardless of browser window size.
  • DON'T REPEAT YOURSELF - Don't repeat CSS code. Instead code once using a class and repeat the class.

CSS PROPERTIES

  • background-color - Sets the background color of an element. Accepts any CSS color unit.
  • background-image - Accepts an image path that applies the file to the background of an element. Background images repeat by defualt. Use "no-repeat" for a single image without repitition.
  • background-repeat- Sets whether or not a background image should be repeated. By default, backgrounds will repeat.
  • border - CSS shorthand property that allows for a visual outline or "border" to be applied to an element. Accepts a length unit to define the border-width, a keyword to define the border-style, and a color unit for the border-color.
  • border-radius - Defines the roundness of a border corner. Accepts any CSS length unit. A value of 100% makes a square picture round (e.g. border-radius-100%;).
  • background-size - Sets the size of background images.
  • clear - The clear property specifies whether an element can be adjacent to floated elements or if it should be moved down and "clear" any floated elements.
  • color - Sets the text color of an element. Accepts any CSS color unit.
  • /* Comment */ MDN: CSS Comments
  • CSS DATA TYPES - a type of value used in CSS See MDN: CSS Data Types Info
  • CSS UNITS FOR LENGTH - See W3 Schools: CSS Units of length.
    • Pixels - When we use pixel units, the size we define will always remain the same and will not scale, regardless of the browser window and size of the screen.
      • Header length will take up 960px of the screen.
  • PERCENTAGES - are always relative to something else. When we use a percentage unit in CSS, the percentage value is measured relative to a parent element's length. MDN: Percentage values
  • display - Sets whether an element should be rendered as a block element, inline element, or inline-block element. Inline-block displays list items horizontally is used to make a menu bar
  • EM - A unit measuring font size and is relative to the font size of the element. One em = 16 pixels. Browser font size is typically 16 pixels by default. Example: "font-size: 1.75em;" makes the font 0.75 larger than the default font of 16px.
  • float - Allows an element to be taken out of the normal flow and placed to the left or right of its container. Text and other inline elements will wrap around it.
  • font-family - Defines which fonts should be applied to text.
  • font-size - Sets the size of text in either relative or absolute units.
  • font-weight - Sets the visual weight of text, which is more commonly known as "bold" as is the case in word processors.
  • list-style - Sets the style of numbers or bullet points for ordered and unordered lists. The marks can also be removed entirely with the value "none".
  • line-height - Set the amount of space between each line of text. Accepts any CSS length unit.
  • margin - Sets the space around the exterior four sides of an element. Accepts any CSS length unit.
  • margin-top and margin-bottom- Create space around an elemnt like a list item or paragraph. Typically, margin-top is 0, and margin-bottom is a value (e.g. "p {margin-top: 0em;margin-bottom: 2em;}").
  • max-width - Sets the maximum width of an element. Accepts any CSS length unit.
  • padding - Sets the space on the interior four sides of an element. Accepts any CSS length unit.
  • text-align - Sets the justification of text, similar to a word processor. For example, text can be set to be aligned to the left, right, or center.
  • text-decoration - Primarily used to add or remove underlines from text.

CSS SELECTORS

  • Class Selector - Selects any HTML element that matches the class and applies a property to them (e.g ".intro {background-color: yellow;}"). W3 Schools on CSS Class

    • In HTML:

    <img src="img/photo1.png" class="reg_img">< /a >

    In CSS:

    .reg_img {
      margin: 0 30px;
    }
  • Pseudo-Class - Dynamic selectors that change based on user interaction with the browser, such as hovering over a link, for example.
    • :link - targets links that have not been visited by the user
    • :visited pseudo-class targets links that have been visited –– or clicked –– by the user
    • :hover pseudo-class targets an element when a user hovers over it.
    • :active pseudo-class gets applied when an element is in an active state.
    • :focus pseudo-class is only applied to interactive elements like links, buttons and form elements. The styles are applied as soon as the element receives focus
  • :nth-child(An+B) - The nth-child pseudo-class matches A(n)+B-1 siblings in the document tree. The A value that precedes n is required. By itself, this will match every "nth" element in a group. Both A and B must be integers. For example, :nth-child(3n+1) will select every 3rd item plus 1. In other words, this will select the 1st item, 4th item, 7th item, 10th item, and so on.

CSS SYNTAX

  • @media - Media queries use the @media CSS rule to define conditions of the browser medium. In other words, it allows for CSS styling to only be included if certain conditions are met, such as a specific browser width. (e.g. "@media screen and (min-width: 480px) {};" and "@media screen and (min-width: 660px);")

Can an element have both an id and a class?

Yes, an element can have one ID (which must be unique!) and multiple classes at the same time. To have multiple classes, use a space between them, here's an example:

<div id="myID" class="class1 class2 class3">Content</div>
