  • JavaScript is case sensitive.
  • ESCAPING THE QUOTES: Sometimes strings contain quotes that are not part of the string. This requires using a back-slash (\) to escape the quotes so the code is clear. 
    var phrase = "He said \"no"\ and left the room."
  • INCREMENT OPERATOR: Shorthand for adding to a variable: To quickly add 10 to the variable "score" use += (or -= /= *=). No spaces between operators. 
    score += 10;
  • Shorthand for adding 1 to a variable: 
    a++;
    Trivia: this is where the name for C++ came from, C + 1, the next version. or to subtract 1 from a varialble... 
    a--;
  • COMMENTS start with //. 
    // This is a comment
    or use /* */ to comment out multiple lines.
  • ASSIGNMENT, EQUALITY, STRICT EQUALITY (=, ==, ===): 
    		= assignment
		== equlity
		=== strict equlity
    • A single equal sign (=) is a command assigning and setting a value to a variable. A double equals (a == 1) is the equality operator and checks a value. Inequality is checked by an exclamation and equal sign (a != 1).
    • Double equals checks to see if the values are the same but is flexible with the type of data. For instance, 123 == "123" will show the number 123 is the same value as the string "123." Same numbers but different type.
    • A common mistake is to use Assignment (=) instead of Equality (==). Assignment is a command, equality is asking if one thing is equal to another. 
      
				var a = 5; //Assigning the value of 5 to variable a
				var b = 10;

				if (a==b) { //Checking if a is equal to b, not a = b
				}
    • THE STRICT EQUALITY OPERATOR (===): Triple equals checks to see if the type and value are exactly the same ( 123 is not equal to the string "123"). It is a good habit to use triple equals when checking equality.
  • CURLY BRACKETS { } groups together a section of code or code block. Usually the opening curly brace is on the same line as the keyword, then the code is written indented on the next line, then the closing curly brace on the following line. 
    
		if (a = b) {
			alert("Yes, a is less than b");
		}
  • NESTING IF STATEMENTS: avoid going more than three layers of nesting "if" statements because code becomes difficult to read and their are more effective methods to use such as functions.
  • LOGICAL AND/OR: Instead of "and" use "&&". Instead of "or" use the vertical bar or pipe symbol "||".
  • SWITCH STATEMENT: Is a good substitute for multiple "if/then/else" statements
    • 
var grade = "Premium"

switch (grade) {
	case "Regular":
			alert("Cost is $3.15");
			break;
	case "Premium":
			alert("Cost is $3.35");	
			break;
	case "Diesel":
			alert("Cost is $3.47");	
			break;
	default:
			alert("That is not a valid grade");
}
